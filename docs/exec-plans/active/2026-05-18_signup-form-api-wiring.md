## 목표

교수(PROF) 회원가입 폼에서 그동안 클라이언트 상태로만 동작하던 두 기능을 실제 서버 API와 연결한다.
1. 이메일 인증번호 받기·확인 → 백엔드 신규 엔드포인트 `/api/auth/email/send`, `/api/auth/email/verify` 호출
2. 닉네임 중복 확인 → 기존 `/api/auth/check-nickname` UI 노출(입력칸 옆 버튼)

## 의사결정 로그

### 2026-05-18
- **결정**: 닉네임 확인을 명시적 버튼 방식으로 구현. **이유**: 사용자가 직접 명시. 디바운스 호출은 사용자가 의도하지 않은 시점에 서버 트래픽을 발생시키며, "확인" 액션이 명확한 사용자 의도를 표현하는 데 더 적합.
- **결정**: 닉네임 입력값 변경 시 이전 확인 결과를 즉시 무효화. **이유**: "abc"에서 통과 후 "abc1"으로 바꿔도 통과로 남으면 중복 검사 우회 가능 — 보안·일관성 문제.
- **결정**: 인증번호 재전송 쿨다운/타이머는 이번 범위에서 제외. **이유**: 백엔드의 재전송 정책(분당 N회, 만료 등)이 아직 명세에 없음. UI에 임의 쿨다운을 박으면 백엔드와 어긋날 수 있어 합의 후 별도 PR.
