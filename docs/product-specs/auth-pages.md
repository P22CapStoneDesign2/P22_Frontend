# 제품 스펙 — 인증 페이지

> 상태: ✅ 구현됨

---

## 로그인 페이지 (`/`)

| 항목 | 내용 |
|------|------|
| 입력 | 이메일, 비밀번호 |
| 성공 | `localStorage`에 토큰 저장 → `/workspace` 이동 |
| 실패 | 에러 메시지 표시 |
| 카카오 로그인 | `/oauth2/authorization/kakao` 링크 (백엔드 리다이렉트) |
| 회원가입 이동 | `/signup` 링크 |

---

## 회원가입 페이지 (`/signup`)

| 항목 | 내용 |
|------|------|
| 입력 | 닉네임, 이메일, 비밀번호, 비밀번호 확인 |
| 성공 | 로그인 페이지(`/`)로 이동 |
| 실패 | 에러 메시지 표시 |
| 유효성 규칙 | 백엔드 스펙 기준: 닉네임 2~20자, 비밀번호 8~20자 영문+숫자+특수문자 |

---

## 카카오 콜백 페이지 (`/oauth2/callback`)

| 항목 | 내용 |
|------|------|
| 진입 | 백엔드가 OAuth2 인증 후 `?accessToken=...&refreshToken=...`으로 리다이렉트 |
| 처리 | URL 쿼리에서 토큰 추출 → `localStorage` 저장 → `getMe()` 호출 |
| 신규 유저 | `username` 없음 → `/oauth2/signup` |
| 기존 유저 | `username` 있음 → `/workspace` |

---

## 카카오 닉네임 입력 페이지 (`/oauth2/signup`)

| 항목 | 내용 |
|------|------|
| 조건 | 카카오 최초 로그인 시에만 표시 |
| 입력 | 닉네임 |
| 성공 | `PATCH /api/users/me`로 닉네임 등록 → `/workspace` 이동 |
